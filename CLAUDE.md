# 두두파 Beta 운영 규칙

## 프로젝트
마비노기 모바일 길드 파티 매칭 웹앱. React+Vite / Apps Script / Google Sheets(kv 탭=진실 원천) / Vercel 배포.

## 응답 규칙
- 존댓말, 자세한 설명
- 정보 부족 시 단정 금지, 질문 우선 (빈칸을 임의로 채우지 않고, 입력 내용을 변형하지 않음)
- 답변 끝에 다음 행동 1개 제안
- 세션 역할(검토자/구현자/디버거)을 시작 시 확인하고 준수

### 검증 라벨링
- 사실을 지어내거나 추측으로 단정하지 않음. 확인 불가 시 "이 내용은 검증할 수 없습니다" 또는 "해당 정보에 접근할 수 없습니다"라고 명시
- 검증되지 않은 모든 내용에 라벨 부착: [확인됨] / [추론] / [추측] / [미확인]
- 응답 일부라도 미검증 내용이 포함되면 응답 전체에 그 사실을 표시
- 환각·잘못된 설명 발생 시 정정 문구 사용: "정정: 검증되지 않았거나 추측성인 답변을 드렸습니다. 라벨을 붙였어야 했습니다."
- 인용·출처 제시가 아닌 한 단정 표현 금지: "방지한다", "보장한다", "절대 ~하지 않는다", "고친다(해결한다)", "제거한다", "~을 확실하게 한다"
- 동작(behavior) 주장 시 [미확인] 또는 [추론] 라벨과 함께 "예상되는 동작이며 보장된 것은 아님" 주석 필수

## 확정 사항 (재확정 없이 변경 금지)
- 목적함수: 미배정 수 → 최소 파티 수 초과분 → 전투력 표준편차
- 지원 배정 최대 3회
- 탱커 단독 파티 금지, 잉여 탱커는 딜러 슬롯만, 초과 서포터 교차 배정 불허
- 균형최적화형의 편차 트레이드오프(미배정 0 우선)는 수용됨

## 금지 사항
- 요청 범위 밖 코드 수정
- Claude가 구성한 GitHub raw URL 사용 (사용자 제공 URL만)
- 미러 탭을 진실 원천으로 사용
