## 1. 신청 유형 (확정)

| 유형 | 배정 규칙 |
|---|---|
| 일반 (normal) | 필수 1회 배정 (전체 기간 중 정확히 1회) |
| 지원 (support) | ~~0~n회~~ → **0~3회 배정** (2026-07-10 변경, 배정되지 않아도 됨) |
| 일반+지원 (both) | 1회 필수 배정 + 배정된 시간을 제외한 나머지 신청 시간에 **0~3회** 지원 배정 |

- 일반+지원 선택은 신청 화면에서 두 버튼을 동시에 선택하는 방식이며, **신청 시간 수 > 신청 캐릭터 수**일 때만 조합 선택이 활성화된다 (사용자 확정).
- "필수 1회"의 의미: 알고리즘이 어떤 경우에도 배정을 강제한다는 뜻이 아니라, 지원 대비 절대 우선이며 배정 실패가 조용히 사라지지 않고 반드시 사유와 함께 미배정 목록으로 관리자에게 보인다는 뜻이다 (10.3.2절 확정).
- ~~1단계 배정에 실패한 both 캐릭터는 신청 시간 전체가 지원 후보로 남는다~~ → **폐기됨 (2026-07-08)**: both 캐릭터는 일반 배정(재시도 포함)이 최종 실패하면 지원 후보에서도 제외된다. "일반 배정이 실패하면 지원도 실패"가 원칙이다.

## 2. 최종 전투력 공식 (확정 — 단, 비율은 추정치)

```
diff        = 캐릭터의 마도저항 − 콘텐츠의 마도압력
보정률      = clamp(0.00015 × diff, −0.40, +0.40)     // 1포인트당 0.015%, 양방향 ±40% 한도
최종 전투력 = max(0, round(기본 전투력 × (1 + 보정률)) − 패널티)
```

- **0.015%/포인트는 추정치다** [Unverified — 게임 공식 데이터로 확정된 값이 아님]. 코드에서 `RESIST_PRESSURE_RATIO` 상수로 분리되어 있어 확정 시 교체 가능하다.
- 마도 압력이 0인 콘텐츠에도 적용된다 (diff = 저항 그대로, 사용자 확정).
- 패널티는 캐릭터 속성이며(캐릭터 관리에서 입력), 어느 파티·콘텐츠·시간대에서든 동일하게 보정 후 차감·0 클램프로 적용된다 (사용자 확정).
- 콘텐츠 맥락이 없는 화면(전체 캐릭터 목록)은 `max(0, 기본 전투력 − 패널티)`로 계산한다.
- 사용자/관리자 두 파일이 동일 공식을 사용함을 검증했다. 다만 공식이 두 파일에 중복 정의되어 있어, 한쪽만 수정되면 다시 불일치가 생길 수 있다 [Inference — 과거 실제로 발생했던 유형의 위험].

## 3. 배정 알고리즘 단계별 정리

균형 범위는 **콘텐츠 전체의 모든 시간대·모든 파티**이며, 탱커·서포터·딜러 **모든 역할**이 균형 대상이다 (사용자 확정). 모든 전투력 비교는 2절의 최종 전투력 기준이다.

### 1단계 — 일반(및 both) 캐릭터의 시간 배정
- 딜러를 먼저 배치해 "딜러 있는 시간" 집합을 만든 뒤, 탱커·서포터는 그 시간을 우선 선택한다 (사용자 확정 — 딜러 없는 시간 회피).
- **딜러는 집중 배치한다 (2026-07-04 정정, 사용자 확정)**: 부하 분산(인원 적은 시간 우선) 대신, 이미 딜러가 있고 파티가 덜 찬 시간대의 빈 딜러 슬롯을 먼저 채워 불필요한 파티(새 시간대) 생성을 줄인다. 운영 시간대 수가 줄어들 수 있다는 트레이드오프가 안내된 뒤 선택된 방향이다.
- 같은 우선순위 안에서는 역할별 시간대 부하 분산(해당 역할 인원이 적은 시간 우선)으로 배정한다.
- 신청 시간이 적은 캐릭터부터 처리한다 (선택지가 좁은 캐릭터 우선) [Inference — 코드에서 확인한 정렬이며, 별도로 확정받은 규칙은 아님].
- 동일 대표 캐릭터는 같은 시간에 1명만 배정된다. 신청한 모든 시간이 충돌로 막힌 캐릭터는 사유와 함께 미배정 목록으로 간다 (사용자 확정: 배정 가능한 1명만 배정, 나머지는 미배정 표시).

### 2단계 — 시간대별 파티 수 산정
- 기본: **파티 수 = ceil(그 시간대의 일반 딜러 수 ÷ 파티당 딜러 슬롯 수)** (사용자 확정). 지원 신청자는 파티 수에 영향을 주지 않는다.
- 딜러 0명 시간대: 일반 탱커가 2명 이상이면 **파티 수 = min(ceil(탱커 ÷ (1+딜러슬롯)), floor(탱커 ÷ 2))** 로 파티를 만들고, 탱커를 딜러 슬롯에도 채용한다 (사용자 확정, 2026-07-04). 탱커 1명 이하면 파티를 만들지 않고 그 시간대의 일반 탱커·서포터를 사유와 함께 미배정 처리한다.
- 일반 신청자가 0명이고 지원만 있는 시간대는 파티를 만들지 않는다 (사용자 확정).

### 3단계 — 시간대별 초기 배치
- 각 역할을 전투력 내림차순으로, 현재 전투력 합이 가장 낮은 파티부터 채운다.
- **역할 초과분 교차 배정 (2026-07-04 재정의, 사용자 확정)**:
  - 딜러 초과분: 탱커 슬롯에 배정 가능
  - 탱커 초과분: 딜러 슬롯에 배정 가능 (남는 슬롯만, 파티 수 미증가 — 기존과 동일)
  - 서포터 초과분: 딜러·탱커 슬롯에 배정 가능. 단 탱커와 달리 2명 이상이어도 파티를 새로 생성하지 않으며, 채울 슬롯이 없으면 미배정 처리
- **빈 슬롯 채우기 방식 (2026-07-04 정정, 사용자 확정)**: 역할 간 우선순위 없이 전투력 순 혼합 — 딜러 슬롯은 [딜러+초과탱커+초과서포터] 혼합 풀, 빈 탱커 슬롯은 남은 [초과딜러+초과서포터] 혼합으로 채운다. (이전의 "본래 역할 우선" 답변은 이 정정으로 대체됨.) 서포터 슬롯은 서포터만 채운다 (사용자 확정). "초과" 기준(탱커·서포터는 본래 슬롯 우선 채움 후 남은 인원, 딜러는 딜러 슬롯에서 밀린 인원)과 "딜러 슬롯 먼저 → 빈 탱커 슬롯" 순서도 사용자 확정.
- **교차 배정 표시**: 슬롯의 role은 그 자리에 배정된 캐릭터의 실제 역할로 표시된다 (예: 딜러 슬롯에 탱커가 교차 배정되면 "탱커"로 표시되어 파티가 탱커2·딜러2·서포터1처럼 보일 수 있음). 이는 구현 세션의 재량 선택으로 확인된 구현 방식이며 [Inference — 코드 정독 기준], 부족 인원 계산은 빈 슬롯의 원래 role 기준으로 동작한다.

### 4단계 — 전역 균형 스왑 (지역 탐색)
- 같은 역할·서로 다른 시간의 배치 쌍을 골라, 교환 시 **전체 파티 평균 전투력의 표준편차**가 줄어들면 교환한다.
- 각 캐릭터는 자신이 실제 신청한 시간 범위 안에서만 이동하며, 동일 대표 동일 시간 충돌은 교환 전에 검사한다.
- 최대 반복 횟수 제한(300회)이 있는 휴리스틱이다. 최적해 보장이 아니다.

### 5단계 — 지원 신청자로 빈자리 채우기
- **(2026-07-04 정정, 사용자 확정)** 지원 후보를 최종 전투력 내림차순으로 순회하며 **전투력 높은 지원자부터 배정**한다. 자리 선택만 균형 기준(채웠을 때 그 파티 평균이 전체 평균에 가장 가까워지는 자리)이다. 이전의 "균형 점수 전역 최소 조합 선택" 방식은 약한 지원자가 우선 배정되는 문제가 관찰되어 대체되었다.
- **미배정 지원자 우선 (사용자 확정)**: 전원이 1회씩 배정될 기회를 가진 뒤에만 같은 캐릭터의 반복(0~n회) 배정을 허용한다.
- ~~지원 교차 배정: 역할 무관, 균형 점수만~~ → **역할 우선 2단계로 대체 (2026-07-08 확정)**: 지원 후보는 본래 역할의 빈 슬롯을 먼저 채우고, 전혀 없을 때만 교차 배정한다 (서포터 슬롯은 서포터만 유지). 균형 점수는 같은 순위 등급 안의 자리 선택에만 사용. 실제 매칭에서 지원 서포터가 빈 서포터 슬롯을 두고 탱커 슬롯에 교차 배정되는 문제가 관찰되어 변경됨.
- 지원 후보는 같은 캐릭터가 여러 시간대에 반복 배정될 수 있고(0~n회), 시간당 동일 대표 1명 제약은 유지된다.
- 남는 지원 신청자는 미배정 목록에 넣지 않아도 된다 (배정 의무 없음, 사용자 확정).
- **지원 라벨**: 지원으로 배정된 슬롯(both 신청자의 지원 배정분 포함)에는 결과 화면에서 "· 지원" 태그가 표시된다. 이미지 다운로드는 같은 화면을 캡처하므로 라벨이 이미지에도 포함될 것으로 예상된다 [Inference — 실행 테스트로 확인된 것은 아님].

### 결과 정리
- 파티별 부족 인원("탱커 1명 부족" 등)을 계산해 표시하고, 시간·파티 번호 순으로 정렬해 반환한다.

## 4. 미배정 처리 원칙

- 일반(및 both) 신청 캐릭터는 어떤 입력에서도 "파티 배정" 또는 "사유가 표시된 미배정" 둘 중 하나로 나타나는 것이 목표이며, 최신 코드 정독에서 조용히 사라지는 경로는 발견되지 않았다 [Inference — 실행 테스트로 검증된 것은 아님].
- 미배정 사유 종류 (코드에서 확인): 동일 대표 캐릭터 시간 충돌 / 역할 자리 부족 / 딜러 없는 시간대(파티 미생성) / 파티 편집 중 제외.
- 미배정 항목의 type은 원 신청 유형(both 포함)을 보존한다.

## 6. 한계와 주의 사항

- 스네이크 드래프트 → "낮은 파티 우선 + 전역 스왑" 구조로 바뀌었지만 여전히 휴리스틱이며, 파티 간 전투력이 완전히 균등해진다고 말할 수 없다.
- 저항-압력 비율(0.015%)은 추정치다 [Unverified]. 게임 공식 확인 시 상수 교체가 필요하다.

## 7. 2026-07-08 개정 — 「일반 신청 파티 배분 로직 수정안」 채택

사용자가 별도 스펙 세션에서 작성한 「일반 신청 파티 배분 로직 수정안」이 채택되었다.
구현은 아직 반영 전이며, 아래 항목이 이 문서의 이전 절 내용을 **대체**한다.

- **딜러 0 시간대 탱커 파티 생성 규칙 폐기 (사용자 확정)**: "탱커 2명 이상이면 파티 생성"
  규칙과 파티 수 공식 min(ceil(탱커÷(1+슬롯)), floor(탱커÷2))는 폐기. 탱커 잉여는 어떤
  경우에도 파티 생성 트리거가 아니며, 딜러 0 시간대는 파티 미생성 + 해당 시간대 전용
  일반 탱커·서포터는 사유와 함께 미배정.
- **서포터 잉여 교차 배치 금지 (재확정)**: 일반 서포터 잉여는 탱커·딜러 슬롯에 배치하지
  않고 미배정 처리. (2026-07-04의 혼합 풀 내 서포터 참여는 폐기됨)
- **탱커 잉여**: 빈 딜러 슬롯만 사용 (딜러를 밀어내지 않음), 불가 시 미배정. 실제 역할 표시 유지.
- **딜러 잉여 → 신규 파티 생성 (신설, 드문 예외용)**: 파티 수 상한은 두지 않으며(사용자
  확정), 딜러 기준 ceil 파티 수 아래에서 이 규칙이 발동하는 시나리오는 희소하다는 분석이
  안내된 뒤 예외 안전장치로 유지가 결정되었다 [Inference — 발동 희소성은 로직 분석 기준
  추정]. 발동 시: ① 지원 서포터 보유 계정 탐색 후 해당 지원 가능 시간에 신규 파티 생성 +
  지원 서포터 배정 ② 불가 시 미배정 딜러 계정이 중복되지 않는 시간(딜러들의 신청 시간 내)에
  신규 파티 생성, 서포터 슬롯은 빈자리 ③ 그래도 불가한 딜러는 사유와 함께 미배정.
  이는 "지원은 파티를 만들지 못한다" 규칙의 명시적 예외다.
- **배치 순서 재정의**: 서포터 우선 배치 → 서포터 포함 누적 전투력 기준 탱커 배치(보완

  계산에 포함.
- **유지**: 딜러 집중 배치(딜러에 한해 수정안의 분산 원칙보다 우선), 지원 강캐우선(미배정
  지원자 우선 1회 후 반복), 일반/지원/both 규칙, 최종 전투력 공식, 파티 수 = 딜러 기준 ceil.
- **[Unverified] 저장소 상태 주의**: 2026-07-08 확인 시 GitHub 저장소 코드는 고도화 버전이
