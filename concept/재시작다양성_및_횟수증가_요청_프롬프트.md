# 버그 수정 요청: 재시작(restart) 다양성 부족 + 재시작 횟수 증가

## 0. 이 문서에 대해

구현 세션에 그대로 붙여넣어 사용하는 문서입니다. 베이스라인은 사용자가 제공하는 최신
`matchEngine_experimental.js`(균형최적화형)입니다. `[Inference]` 표시는 확정 사실이 아닌
추론이며, 구현 전 애매하면 사용자에게 확인해야 합니다. 이 문서의 서술은 코드 정독
기준이며 실행 검증된 것이 아닙니다.

범위: **초기해 생성(`buildInitialState`)의 재시작 다양성**과 **재시작 횟수(`RESTARTS`)**만
변경합니다. 힐클라이밍 루프 자체(`localSearch`, `objectiveOf`, 이웃 탐색 연산자), 지원
채우기(8절), 3회 제한은 변경하지 않습니다.

---

## 1. 배경: 실측으로 확인된 문제

실제 구글 시트 데이터(20260709_test, 27계정)로 균형최적화형 결과를 확인한 결과, 파티 간
전투력 편차(표준편차 약 2,610, 범위 68,990~78,343)가 관찰되었다. 원인을 추적한 결과 지원
채우기 회귀는 이미 수정되어 있었고(전역 균형·3회 제한 모두 정상 반영 확인됨), 남은 원인은
**재시작이 실질적으로 같은 초기해를 반복 생성한다는 것**이었다.

`buildInitialState`의 정렬 비교 함수:

```js
const dealerOrder = [...dealerEntries].sort((a, b) =>
  a.times.length - b.times.length || (shuffleSeed ? Math.random() - 0.5 : 0));
```

`Math.random()`은 **신청 시간 개수(`times.length`)가 정확히 같은 캐릭터끼리의 동률
처리에서만** 작동한다. 신청 시간 개수가 서로 다른 캐릭터 간의 상대 순서는 재시작 여부와
무관하게 항상 고정된다. `otherOrder`(탱커·서포터)도 동일한 패턴이다. 그 결과
`RESTARTS`(현재 3회)가 서로 의미 있게 다른 초기해를 거의 만들어내지 못하고, 힐클라이밍이
같은 지역 최적에 반복해서 도달할 가능성이 높다.

## 2. 확정된 수정 사항 (사용자가 직접 결정 — 추측 아님, 2026-07-10)

### 2.1 재시작마다 처리 순서를 온전히 무작위화

`shuffleSeed`(재시작 여부)가 참일 때, 정렬을 "시간 개수 우선 + 동률만 무작위"가 아니라
**매 재시작마다 처리 순서 자체를 완전히 무작위로 섞는 방식**으로 바꾼다.

`[Inference]` 구현 형태 제안 (세부는 구현자 재량, 결과 기준은 2.3절):
- 안정적인 첫 시도(`shuffleSeed`가 거짓, 즉 첫 restart)는 **현재처럼 결정적 정렬**(신청
  시간 적은 순)을 유지한다 — 이건 딜러 집중 배치 등 기존에 검증된 좋은 출발점이므로 그대로
  둔다.
- 이후 재시작(`shuffleSeed`가 참인 회차)부터는, 정렬 기준(시간 개수 오름차순) 자체는
  유지하되 그 정렬을 "완전히 깨는" 것이 아니라, **배열 전체에 무작위성을 실질적으로
  주입**한다. 예: 정렬 후 인접 원소들을 일정 확률로 뒤섞거나(controlled shuffle), 혹은
  아예 시간 개수 우선순위를 낮은 가중치로만 반영하고 대부분 무작위 순서로 처리하는 등 —
  핵심은 "신청 시간 개수가 다른 캐릭터끼리도 재시작마다 처리 순서가 바뀔 수 있어야 한다"는
  것이다.
- `dealerOrder`, `otherOrder` 양쪽 모두 동일하게 수정한다.

### 2.2 재시작 횟수 증가

`RESTARTS`를 **10회**로 늘린다 (기존 3회 → 10회, `aggressive` 모드는 별도 — 기존 5회에서
비례해 늘릴지는 `[Inference — 확인 필요]`, 늘리지 않고 그대로 둬도 무방하다).

```js
const RESTARTS = aggressive ? 5 : 10; // 또는 aggressive도 비례 조정, 재량
```

### 2.3 완료 기준

- 재시작마다 `buildInitialState`가 만드는 초기해가 서로 유의미하게 달라야 한다 (같은 입력에
  대해 10번 재시작했을 때 최종 배치가 매번 동일하게 수렴하지 않아야 함 — 완전히 같은 결과가
  나온다면 무작위화가 아직 부족한 것).
- `RESTARTS = 10`으로 동작한다.
- 재시작 다양성 개선 후, 20260709_test와 같은 실데이터로 다시 실행했을 때 파티 간 전투력
  표준편차가 기존(약 2,610)보다 개선되는 경향을 보인다 `[Inference — 기대 효과이며, 신청자
  풀 자체의 전투력 격차로 인한 하한선은 있을 수 있음. 절대적 수치 목표를 정하는 것이
  아니라 개선 방향만 확인]`.
- 딜러 집중 배치 등 첫 번째(비무작위) 시도의 품질은 그대로 유지된다 — 재시작 다양성 개선이
  기존에 검증된 좋은 출발점을 훼손하지 않는다.
- 기존 규칙(목적함수 우선순위, 이웃 탐색 연산자, 지원 채우기, 3회 제한 등)은 변경 없음.
