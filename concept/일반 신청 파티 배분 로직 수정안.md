# 일반 신청 파티 배분 로직 수정안

## 1. 적용 범위

이번 변경은 기존 자동 매칭 로직 중 다음 영역에만 적용합니다.

* 일반 신청 캐릭터의 파티 배치 순서
* 일반 신청 배치 후 역할별 잉여 캐릭터 처리
* 딜러 잉여 발생 시 신규 파티 생성
* 신규 파티 생성 시 지원 서포터 탐색
* 지원 서포터를 구할 수 없는 경우 빈 서포터 슬롯 생성

다음 기존 규칙은 유지합니다.

* 일반 신청 캐릭터는 전체 신청 시간 중 최대 1회 배정
* 지원 신청 캐릭터는 서로 다른 시간대에 0~N회 배정 가능
* 일반+지원 신청 규칙
* 동일 대표 캐릭터는 같은 시간대에 한 캐릭터만 배정
* 신청하지 않은 시간에는 배정 불가
* 일반 신청이 지원 신청보다 우선
* 최종 전투력을 기준으로 파티 간 균형 조정
* 동일 역할 및 배치 가능한 캐릭터 간 전역 균형 스왑
* 관리자의 매칭 결과 수동 수정
* 일반 신청 미배정자는 사유와 함께 미배정 목록 표시

---

# 2. 일반 신청 캐릭터의 시간 배정

일반 신청 캐릭터를 실제 시간대에 배정하는 기존 로직은 최대한 유지합니다.

각 캐릭터는 자신이 신청한 시간 중 하나에만 일반 신청자로 배정됩니다.

시간 배정 시 다음 조건을 유지합니다.

1. 동일 대표 캐릭터는 같은 시간대에 한 캐릭터만 배정
2. 신청 시간이 적어 선택지가 좁은 캐릭터를 우선 고려
3. 역할별 시간대 인원이 과도하게 집중되지 않도록 분산
4. 캐릭터의 역할이 필요한 시간대를 우선 고려
5. 배정할 수 없는 일반 신청 캐릭터는 미배정 사유 표시

시간대가 결정된 이후, 각 시간대 안에서 아래의 변경된 파티 배치 순서를 적용합니다.

---

# 3. 일반 신청 파티 배치 순서

## 3.1 1순위: 서포터 배치

각 시간대에서 일반 신청 서포터를 가장 먼저 파티에 배치합니다.

기본 원칙:

* 파티당 서포터 슬롯은 1개
* 각 파티에 서포터를 최대 1명씩 배치
* 서포터가 여러 명이면 파티별로 분산 배치
* 파티 간 전투력 균형을 고려하여 배치
* 서포터가 없는 파티는 서포터 슬롯을 빈자리로 유지

서포터는 파티의 기본 구성 조건이므로 탱커와 딜러보다 우선 배치합니다.

## 3.2 2순위: 탱커 배치

서포터 배치가 끝난 뒤 탱커를 배치합니다.

탱커 배치 기준:

1. 서포터가 배치된 파티의 현재 전투력 확인
2. 파티 간 예상 평균 전투력이 최대한 균일해지도록 탱커 배치
3. 파티당 탱커 기본 슬롯은 1개
4. 동일 대표 캐릭터의 같은 시간 중복 여부 확인

예시:

* 서포터 전투력이 높은 파티에는 상대적으로 낮은 전투력의 탱커 배치
* 서포터 전투력이 낮은 파티에는 상대적으로 높은 전투력의 탱커 배치

단순히 탱커를 전투력 순서대로 넣는 것이 아니라, 서포터까지 포함한 파티의 누적 전투력을 기준으로 배분합니다.

## 3.3 3순위: 딜러 배치

서포터와 탱커 배치가 끝난 뒤 딜러를 배치합니다.

딜러 배치 기준:

1. 현재 파티 전체 전투력 합계 확인
2. 전투력 합계가 낮은 파티부터 딜러 배치
3. 파티별 딜러 평균 전투력이 최대한 비슷하도록 조정
4. 파티 전체 평균 전투력 균형을 딜러 균형보다 우선
5. 딜러 슬롯 수를 초과하지 않도록 배치

딜러 배치 후 기존 전역 균형 스왑 로직을 실행할 수 있습니다.

# 4. 일반 신청 배치 완료 후 잉여 캐릭터 처리

## 4.1 서포터 잉여

파티별 서포터 슬롯을 모두 채운 뒤에도 일반 신청 서포터가 남는 경우입니다.

처리 방식:

* 서포터는 다른 역할 슬롯에 교차 배치하지 않음
* 신규 파티 생성의 기준으로 사용하지 않음
* 남은 서포터는 미배정 처리
* 미배정 목록에 사유 표시

미배정 사유 예시:

> 배정 가능한 서포터 자리가 없습니다.

이번 변경 이후 서포터는 다음 슬롯에 배정할 수 없습니다.

* 탱커 슬롯
* 딜러 슬롯

서포터가 파티 배치의 최우선 역할이기 때문에, 기존 문서의 ‘초과 서포터를 탱커 또는 딜러 슬롯에 교차 배치’하는 규칙은 일반 신청 단계에서는 사용하지 않습니다.

지원 신청 서포터 역시 기본적으로 빈 서포터 슬롯을 채우는 용도로 우선 사용합니다.

## 4.2 탱커 잉여

파티별 탱커 슬롯을 모두 채운 뒤 일반 신청 탱커가 남는 경우입니다.

처리 우선순위:

1. 비어 있는 딜러 슬롯에 탱커 배치 검토
2. 파티 전체 전투력 균형이 개선되는 경우 딜러 슬롯에 교차 배치
3. 배치 가능한 딜러 슬롯이 없으면 미배정 처리
4. 관리자가 결과 화면에서 직접 추가 조정

탱커를 딜러 슬롯에 배치할 때는 캐릭터의 실제 역할을 유지합니다.

표시 예시:

* 해당 슬롯의 기본 자리: 딜러
* 배정 캐릭터 역할: 탱커
* 결과 화면 표시: 탱커

따라서 한 파티가 다음과 같이 보일 수 있습니다.

* 탱커 2명
* 서포터 1명
* 딜러 1명

탱커 잉여만을 이유로 신규 파티를 자동 생성하지 않습니다.

미배정 사유 예시:

> 배정 가능한 탱커 또는 딜러 자리가 없습니다.

---

# 5. 딜러 잉여 처리

딜러 슬롯을 모두 채운 뒤 일반 신청 딜러가 남는 경우, 남은 딜러를 단순 미배정하지 않고 신규 파티 생성을 우선 검토합니다.

## 5.1 신규 파티 생성 가능 조건

다음 조건을 충족하는 경우 신규 파티 생성 대상으로 처리합니다.

* 일반 신청 딜러가 기존 파티에 배정되지 못함
* 해당 딜러가 신청한 시간 중 배정 가능한 시간이 존재함
* 동일 대표 캐릭터가 해당 시간에 다른 캐릭터로 배정되어 있지 않음
* 같은 시간에 배정할 딜러가 한 명 이상 존재함

신규 파티는 딜러 잉여가 존재할 때만 생성합니다.

서포터 잉여 또는 탱커 잉여만으로 신규 파티를 생성하지 않습니다.

---

# 6. 딜러 잉여 신규 파티 생성 1단계

## 6.1 지원 서포터 보유 계정 탐색

미배정 딜러를 배치할 신규 파티를 만들기 전에, 서포터 캐릭터를 지원 신청한 대표 캐릭터를 탐색합니다.

탐색 대상:

* 지원 신청 또는 일반+지원 신청 계정
* 하위 캐릭터 중 서포터 역할 캐릭터 보유
* 해당 서포터 캐릭터가 지원 후보로 등록되어 있음
* 신규 파티를 만들 시간에 지원 가능
* 동일 대표 캐릭터가 해당 시간에 이미 배정되어 있지 않음

## 6.2 지원 서포터 기준 시간 탐색

지원 서포터가 참여 가능한 시간 중에서 미배정 딜러 계정들이 배정 가능한 시간을 찾습니다.

후보 시간이 여러 개라면 다음 기준으로 선택합니다.

1. 배정 가능한 미배정 딜러 수가 가장 많은 시간
2. 서로 다른 대표 캐릭터의 딜러를 가장 많이 배치할 수 있는 시간
3. 기존 파티 및 신규 파티 전체의 전투력 균형이 좋아지는 시간
4. 지원 서포터의 다른 시간대 활용 가능성을 과도하게 제한하지 않는 시간

## 6.3 신규 파티 구성

적합한 시간이 발견되면 해당 시간에 신규 파티를 생성합니다.

배치 순서:

1. 지원 가능한 서포터 캐릭터 배치
2. 미배정 일반 딜러 배치
3. 필요한 경우 배정 가능한 일반 탱커 잉여 캐릭터 검토
4. 남은 자리는 빈 슬롯으로 유지

신규 파티 예시:

* 탱커: 빈 슬롯
* 서포터: 지원 서포터 A
* 딜러: 미배정 딜러 B
* 딜러: 미배정 딜러 C

지원 서포터에는 결과 화면에서 지원 표시를 적용합니다.

## 6.4 동일 대표 캐릭터 제한

신규 파티에서도 다음 조건을 반드시 적용합니다.

* 같은 대표 캐릭터의 캐릭터는 동일 시간에 한 명만 배정
* 지원 서포터의 대표 캐릭터가 해당 시간에 이미 배정되어 있으면 사용 불가
* 미배정 딜러의 대표 캐릭터가 해당 시간에 이미 배정되어 있으면 해당 딜러는 사용 불가

# 7. 딜러 잉여 신규 파티 생성 2단계

지원 가능한 서포터를 찾을 수 없는 경우, 미배정 딜러만으로 신규 파티 생성을 검토합니다.

## 7.1 공통 배정 가능 시간 탐색

미배정 딜러 캐릭터들의 신청 시간을 비교하여, 각 대표 캐릭터가 중복되지 않게 배정할 수 있는 시간을 찾습니다.

시간 후보별로 다음을 계산합니다.

* 해당 시간에 배정 가능한 미배정 딜러 수
* 서로 다른 대표 캐릭터 수
* 동일 대표 캐릭터 시간 중복 발생 여부
* 기존 파티와의 전투력 균형
* 신규 파티의 예상 딜러 평균 전투력

## 7.2 신규 파티 시간 선택 기준

후보 시간이 여러 개라면 다음 순서로 결정합니다.

1. 서로 다른 대표 캐릭터의 딜러를 가장 많이 배치할 수 있는 시간
2. 미배정 딜러 수를 가장 많이 줄일 수 있는 시간
3. 파티 전체 전투력 균형이 좋아지는 시간
4. 딜러 전투력 균형이 좋아지는 시간

## 7.3 신규 파티 생성

선택한 시간에 신규 파티를 생성하고 미배정 딜러들을 배치합니다.

이 경우 지원 가능한 서포터를 찾지 못한 상태이므로 서포터 자리는 빈 슬롯으로 생성합니다.

예시:

* 탱커: 빈 슬롯
* 서포터: 빈 슬롯
* 딜러: 미배정 딜러 A
* 딜러: 미배정 딜러 B

부족 인원 표시:

* 탱커 1명 부족
* 서포터 1명 부족

---

# 8. 딜러 잉여 신규 파티 생성 반복

미배정 딜러가 파티당 딜러 슬롯 수보다 많은 경우 신규 파티 생성을 반복합니다.

처리 순서:

1. 지원 서포터와 함께 생성할 수 있는 파티를 우선 생성
2. 지원 서포터를 사용할 수 없는 딜러들은 중복되지 않는 시간에 신규 파티 생성
3. 서포터를 구하지 못한 신규 파티는 서포터 빈 슬롯 유지
4. 더 이상 신규 파티에 배치할 수 없는 딜러는 미배정 처리

반복 종료 조건:

* 모든 일반 딜러가 배정됨
* 남은 딜러 사이에 중복되지 않는 배정 가능 시간이 없음
* 동일 대표 캐릭터 시간 충돌로 배치 불가
* 신청한 시간에 신규 파티를 만들 수 없음

미배정 사유 예시:

> 다른 신청 캐릭터와 시간대가 중복되어 신규 파티에 배정할 수 없습니다.

또는:

> 신청한 시간에 배정 가능한 신규 파티를 생성할 수 없습니다.

# 9. 변경 후 전체 매칭 처리 순서

## 9.1 일반 신청 시간 배정

* 일반 및 일반+지원 캐릭터의 일반 배정 시간 결정
* 동일 대표 캐릭터 시간 중복 검사
* 배정 실패 캐릭터 미배정 사유 기록

## 9.2 기존 파티 틀 생성

* 기존 딜러 기준 파티 수 산정 로직을 최대한 유지
* 시간대별 기본 파티 틀 생성

## 9.3 일반 신청 서포터 배치

* 각 파티에 서포터 최대 1명
* 전투력 균형을 고려하여 분산
* 잉여 서포터는 미배정

## 9.4 일반 신청 탱커 배치

* 서포터가 배치된 상태의 파티 전투력을 기준으로 배분
* 각 파티에 기본 탱커 최대 1명

## 9.5 일반 신청 딜러 배치

* 파티 전체 전투력 균형 우선
* 딜러 평균 전투력 균형 차순위

## 9.6 탱커 잉여 처리

* 빈 딜러 슬롯 교차 배치
* 배치 불가능하면 미배정

## 9.7 서포터 잉여 처리

* 다른 역할 슬롯에 교차 배치하지 않음
* 미배정 처리

## 9.8 딜러 잉여 처리

1. 지원 가능한 서포터 계정 탐색
2. 지원 서포터 시간과 딜러 배정 가능 시간을 비교
3. 지원 서포터를 포함한 신규 파티 생성
4. 지원 서포터를 찾지 못하면 딜러 계정이 중복되지 않는 시간에 신규 파티 생성
5. 서포터 슬롯은 빈자리로 생성
6. 더 이상 배정 불가능한 딜러는 미배정

## 9.9 전투력 균형 조정

* 기존 전역 균형 스왑 적용
* 신규 파티도 균형 계산 대상에 포함
* 역할 구성과 대표 캐릭터 시간 중복 조건을 깨는 스왑 금지

## 9.10 잔여 지원 신청 배치

신규 파티 생성 과정에서 사용되지 않은 지원 신청자는 기존 규칙대로 남은 빈자리에 배치합니다.

우선 검토 대상:

1. 빈 서포터 슬롯에 지원 서포터
2. 빈 탱커 또는 딜러 슬롯에 배정 가능한 지원 캐릭터
3. 전체 전투력 균형 개선 여부

지원 신청자는 배정되지 않아도 미배정 목록에 표시하지 않습니다.

## 9.11 최종 검증

* 동일 대표 캐릭터 동일 시간 중복
* 동일 캐릭터 동일 시간 중복
* 일반 신청 캐릭터 다중 시간 배정
* 신청하지 않은 시간 배정
* 파티 정원 초과
* 역할 슬롯 상태
* 필요 마도 저항 미달
* 미배정 일반 신청자의 사유 누락
* 지원 서포터의 동일 시간 중복

# 10. 기존 문서에서 변경되는 규칙

## 10.1 일반 신청 초기 배치 순서

기존:

* 역할별 캐릭터를 전투력 순으로 각 파티에 배치
* 빈 슬롯에 역할 초과 인원을 혼합 배치

변경:

1. 서포터 우선 배치
2. 서포터가 포함된 파티 전투력에 맞춰 탱커 배치
3. 파티 누적 전투력에 맞춰 딜러 배치

## 10.2 서포터 초과분

기존:

* 서포터 초과분을 딜러 또는 탱커 슬롯에 교차 배치 가능

변경:

* 일반 신청 서포터 초과분은 교차 배치하지 않음
* 서포터 슬롯이 없으면 미배정 처리

## 10.3 탱커 초과분

기존:

* 딜러 슬롯에 교차 배치 가능

변경:

* 기존 규칙 유지
* 딜러 슬롯에 배치하거나 미배정
* 신규 파티 자동 생성 기준으로 사용하지 않음

## 10.4 딜러 초과분

기존:

* 탱커 슬롯에 교차 배치하거나 미배정

변경:

* 기존 파티의 배정 가능한 슬롯을 먼저 사용
* 남는 일반 딜러가 있으면 신규 파티 생성 검토
* 지원 가능한 서포터를 우선 탐색해 신규 파티 생성
* 지원 서포터가 없으면 미배정 딜러 계정이 중복되지 않는 시간에 신규 파티 생성
* 해당 신규 파티의 서포터 슬롯은 빈자리 처리

```text
handleSurplusDealers()

findSupportAccountsForNewParty()
findBestTimeForSupportParty()
findBestNonConflictingDealerTime()
createOverflowDealerParty()

validateOwnerTimeConflict()

buildUnassignedReason()
```

## 11.1 딜러 잉여 처리 함수 예시

```text
handleSurplusDealers(unassignedDealers, supportCandidates, parties)
  1. 아직 배정되지 않은 일반 딜러 목록 확인
  2. 지원 서포터를 사용할 수 있는 시간 탐색
  3. 가능한 경우 서포터 포함 신규 파티 생성
  4. 불가능한 경우 딜러끼리 중복되지 않는 시간 탐색
  5. 해당 시간에 서포터 빈 슬롯 신규 파티 생성
  6. 배정된 딜러를 미배정 목록에서 제거
  7. 더 이상 배정할 수 없을 때까지 반복
  8. 남은 딜러에 미배정 사유 기록
```

# 12. 예시

## 12.1 지원 서포터를 활용할 수 있는 경우

미배정 딜러:

* 딜러 A: 20:00, 21:00
* 딜러 B: 21:00, 22:00

지원 서포터:

* 서포터 C: 21:00 지원 가능

결과:

21:00 신규 파티

* 탱커: 빈 슬롯
* 서포터: 서포터 C · 지원
* 딜러: 딜러 A
* 딜러: 딜러 B

## 12.2 지원 서포터를 활용할 수 없는 경우

미배정 딜러:

* 딜러 A: 20:00, 21:00
* 딜러 B: 21:00, 22:00

지원 가능한 서포터 없음

결과:

21:00 신규 파티

* 탱커: 빈 슬롯
* 서포터: 빈 슬롯
* 딜러: 딜러 A
* 딜러: 딜러 B

부족 인원:

* 탱커 1명 부족
* 서포터 1명 부족

## 12.3 동일 대표 캐릭터가 중복되는 경우

미배정 딜러:

* 딜러 A / 대표 캐릭터 X: 21:00
* 딜러 B / 대표 캐릭터 X: 21:00
* 딜러 C / 대표 캐릭터 Y: 21:00

21:00 신규 파티에는 대표 캐릭터 X의 딜러 중 한 명만 배정할 수 있습니다.

결과:

* 딜러 A 또는 B 중 한 명 배정
* 딜러 C 배정
* 나머지 한 명은 다른 신청 시간이 없으면 미배정

미배정 사유:

> 동일 대표 캐릭터의 다른 캐릭터가 해당 시간에 이미 배정되어 있습니다.
