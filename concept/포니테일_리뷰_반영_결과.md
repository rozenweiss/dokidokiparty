## 생성된 파일

### `src/lib/constants.js` (신규)
순수 JS 데이터 상수. React / lucide-react 의존성 없음.

| 상수 | 설명 |
|------|------|
| `DEFAULT_JOBS` | 기본 직업 목록 (8종) |
| `DEFAULT_CONTENTS` | 기본 콘텐츠 목록 (3종, Matcher 기준으로 통합) |
| `ROLE_LABEL` | `{ tank: "탱커", support: "서포터", dealer: "딜러" }` |

> `APP_TYPE_LABEL`은 두 화면 표기가 달라(`"일반 신청"` vs `"일반"`) 각자 유지.
> `ROLE_ICON`은 `lucide-react` 의존성 때문에 공유 불가 → 각 파일에 유지.

---

### `src/lib/utils.js` (신규)
순수 함수 유틸리티. React / lucide-react / storage 의존성 없음.

| 함수/상수 | 설명 |
|-----------|------|
| `timeSlots(start, end, interval)` | 시간 슬롯 목록 생성 |
| `RESIST_PRESSURE_RATIO`, `RESIST_PRESSURE_CAP` | 마도압력 보정 상수 [Unverified — 추정치] |
| `charFinalPower(char, content)` | 최종 전투력 계산 (저항-압력 보정 + 패널티 차감) |
| `groupCandidatesByChar(candidates)` | 후보 목록을 캐릭터 단위로 묶기 |
| `stdev(nums)` | 표준편차 |
| `appliesNormal(type)` | `type === "normal" \| "both"` |
| `appliesSupport(type)` | `type === "support" \| "both"` |

---

### `src/GuildPartyMatcher.css` (신규)
`GuildPartyMatcher.jsx`의 `GlobalStyle` 인라인 CSS 추출. `.gpm-*` 클래스 전용.

### `src/GuildPartyMatcherAdmin.css` (신규)
`GuildPartyMatcherAdmin.jsx`의 `GlobalStyle` 인라인 CSS 추출. `.gpa-*` 클래스 전용.

## 주의 사항

- **`DEFAULT_CONTENTS` 통합**: Admin은 `c1` 하나만 있었으나 constants.js는 Matcher의 3종으로 통합. 길드 설정이 storage에 없을 때만 시드 데이터가 쓰이므로 영향 범위 낮음.
- **CSS 빌드 경고**: `GuildPartyMatcher.css`의 `.gpm-checkbox:checked::after { content: "✓"; ... }`에서 따옴표를 CSS 파서가 unbalanced `{`로 잘못 인식하는 경고가 출력됨. 실제 렌더링에는 무영향이며 빌드는 정상 완료됨.
- **re-export 패턴**: `matchEngine.js`는 기존 import 경로를 깨지 않기 위해 utils.js의 함수를 re-export함.
