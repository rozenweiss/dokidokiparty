# 요청: 지원 채우기 슬롯 선택을 역할 우선으로 변경 (수정안 9.10 정합화)

## 0. 이 문서에 대해

이 프롬프트는 구현 세션에 그대로 붙여넣어 사용하기 위해 작성되었습니다. 베이스라인은
「일반 신청 파티 배분 로직 수정안」이 반영된 **최신 소스**이며, 사용자가 파일을 직접
제공합니다. `[Inference]` 표시는 확정 사실이 아닌 추론입니다. 이 문서의 동작 서술은
스크린샷·코드 분석 기준의 기대 동작이며 실행 테스트로 검증된 것이 아닙니다.

범위: **지원 채우기(잔여 지원 신청 배치)의 슬롯 선택 우선순위만** 변경합니다.
강캐우선 패스1/패스2 구조, 일반 배정 로직, 신규 파티 생성, 재시도, both 연동 실패 규칙,
최종 전투력 공식은 변경하지 않습니다.

## 1. 배경: 관찰된 문제 (실제 매칭 결과)

21:50 시간대에서 지원 서포터 캐릭터가 파티 2·3의 **빈 서포터 슬롯을 두고**, 파티 1의
**탱커 슬롯에 교차 배정**되었다. 결과적으로 파티 1에는 서포터 2명이 뭉치고 탱커가 없으며,
파티 2·3은 "서포터 1명 부족"으로 남았다.

원인 `[Inference — 규칙 분석 기준]`: 현재 지원 채우기의 슬롯 선택이 "역할 우선순위 없음 +
균형 점수(파티 평균의 전체 평균 근접)만"으로 동작하기 때문. 균형 수치상 최선의 자리가
본래 역할 슬롯이 아닐 수 있고, 그 경우 위와 같은 부자연스러운 구성이 나온다.

## 2. 확정된 새 규칙 (사용자가 직접 결정 — 추측 아님, 2026-07-08)

지원 채우기의 슬롯 선택을 **역할 우선 2단계**로 변경한다 (수정안 9.10의 우선 검토 순서를
따름):

- **1순위: 본래 역할의 빈 슬롯.** 지원 후보는 자기 역할과 같은 role의 빈 슬롯이 (자신이
  배정 가능한 시간·제약 안에서) 하나라도 있으면 반드시 그중에서 고른다.
- **2순위: 교차 슬롯.** 본래 역할 빈 슬롯이 전혀 없을 때만 교차 배정한다.
  - 서포터 후보 → 빈 탱커·딜러 슬롯
  - 탱커 후보 → 빈 딜러 슬롯
  - 딜러 후보 → 빈 탱커 슬롯
  - 서포터 슬롯은 어느 경우에도 서포터 후보만 채운다 (기존 확정 유지)
- **균형 점수는 같은 순위 등급 안에서 자리를 고르는 기준**으로만 사용한다 (채웠을 때 파티
  평균이 전체 평균에 가장 가까워지는 자리).
- 강캐우선(전투력 내림차순, 패스1: 미배정 지원자 전원 1회 → 패스2: 반복)은 그대로 유지하며,
  각 후보의 차례에서 위 2단계 규칙으로 자리를 고른다.

이 변경으로 이전 확정이었던 "역할 우선순위 없음, 균형 점수만"(2026-07-04)은 **대체**된다.

## 3. 완료 기준

- 1절의 시나리오 재현 시: 지원 서포터는 배정 가능한 빈 서포터 슬롯이 존재하는 한 반드시
  서포터 슬롯에 배정된다 (탱커 슬롯 교차 금지).
- 본래 역할 빈 슬롯이 전혀 없는 경우에만 교차 배정이 일어난다.
- 서포터 슬롯에 탱커·딜러 후보가 배정되는 경우가 없다.
- 강캐우선 패스1/패스2, 시간당 동일 대표 1명, 신청 시간 범위, "· 지원" 라벨, both 연동 실패
  규칙(일반 실패 시 지원 제외)이 모두 유지된다.
- 더미 데이터로 1절 유형(빈 서포터 슬롯 + 지원 서포터 1명 + 빈 탱커 슬롯 공존)을 실행해
  서포터 슬롯이 먼저 채워지는지 확인한다.
