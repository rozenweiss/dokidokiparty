# 포니테일 리뷰 및 감사(Audit) 반영 요청 프롬프트

현재 코드베이스 전반에 존재하는 과도한 설계(over-engineering), 중복 코드 및 불필요한 커스텀 구현을 네이티브 기능으로 교체하여 코드를 간소화해주세요. 
아래 항목들을 각각 수정하고 불필요한 코드는 삭제해야 합니다.

## 1. 극심한 코드 중복 해소 (상수 및 유틸리티 분리)
- **대상 파일:** `src/GuildPartyMatcher.jsx`, `src/GuildPartyMatcherAdmin.jsx`, `src/lib/matchEngine.js`, `src/lib/matchEngine.experimental.js`
- **내용:** 두 프론트엔드 파일과 매칭 엔진 간에 중복된 코드(`DEFAULT_JOBS`, `DEFAULT_CONTENTS`, `ROLE_LABEL`, `ROLE_ICON`, `APP_TYPE_LABEL`, `timeSlots`, `charFinalPower`)를 `src/lib/constants.js` 및 `src/lib/utils.js`로 분리하여 한 곳에서 `import` 하도록 수정하세요. 매칭 엔진에 중복된 `groupCandidatesByChar`, `stdev` 함수도 포함됩니다.

## 2. 기괴한 패치 스크립트 삭제
- **대상 파일:** `patch.cjs`
- **내용:** 소스 코드를 정규식으로 강제 수정하는 250줄짜리 Node.js 스크립트는 명백한 오버엔지니어링입니다. 스크립트를 삭제하고, 변경 사항이 필요하다면 직접 소스 코드에 반영하거나 Git 커밋을 활용하세요.

## 3. 커스텀 `uid()` 함수 대체
- **대상 파일:** `src/GuildPartyMatcher.jsx`, `src/GuildPartyMatcherAdmin.jsx`
- **내용:** `uid` 생성 함수(`Math.random().toString(36)...`)를 삭제하고, 해당 기능을 호출하는 부분을 모두 브라우저 네이티브인 `crypto.randomUUID()`로 대체하세요.

## 4. 커스텀 `Checkbox` 컴포넌트 네이티브화
- **대상 파일:** `src/GuildPartyMatcher.jsx`
- **내용:** `div` 태그로 억지로 구현한 `Checkbox` 컴포넌트를 삭제하고, 네이티브 `<input type="checkbox" />` 요소를 사용하세요. 스타일링은 CSS의 `accent-color` 속성 또는 `:checked` 가상 클래스를 활용하세요.

## 5. `JobCombo` 컴포넌트를 네이티브 요소로 교체
- **대상 파일:** `src/GuildPartyMatcher.jsx`
- **내용:** 수동 외부 클릭 감지 로직, `useRef` 등 복잡한 상태 관리가 포함된 커스텀 `JobCombo` 컴포넌트 전체를 지우고, 네이티브 `<input list="job-list">`와 `<datalist id="job-list">`로 단순화하세요. 

## 6. 무거운 커스텀 모달과 상태 관리 간소화
- **대상 파일:** `src/GuildPartyMatcherAdmin.jsx` (`ConfirmModal`)
- **내용:** 삭제 확인창을 띄우기 위해 만들어진 각종 `useState` 상태와 `ConfirmModal` 컴포넌트를 제거하세요. 브라우저 네이티브인 `window.confirm()`을 사용하거나, 꼭 필요하다면 네이티브 `<dialog>` 태그로 간소화하여 불필요한 상태 관리를 없애세요.

## 7. 스타일(CSS) 복붙 제거
- **대상 파일:** `src/GuildPartyMatcher.jsx`, `src/GuildPartyMatcherAdmin.jsx`
- **내용:** 두 파일 상단에 `GlobalStyle`로 중복 선언된 거대한 인라인 CSS를 삭제하고, 공통 `src/index.css`나 `src/styles.css` 파일로 추출하여 재사용하세요.

## 8. 깊은 복사(Deep Clone) 로직 개선
- **대상 파일:** `src/lib/matchEngine.experimental.js`
- **내용:** `cloneState` 내의 장황한 수동 복사 로직을 네이티브 API인 `structuredClone(s)`로 교체하거나, 복사가 아예 필요 없는 인플레이스(in-place) 방식으로 수정하세요.

## 9. 과도한 알고리즘 파일 폐기 (선택 사항)
- **대상 파일:** `src/lib/matchEngine.experimental.js`
- **내용:** 소규모 길드 파티 매칭에 힐 클라이밍(Hill-Climbing) 등 복잡한 탐색 알고리즘을 도입하는 것은 불필요한 오버엔지니어링입니다. `matchEngine.stable.js`의 그리디 방식으로 충분하므로, 해당 실험적 파일은 삭제를 강력히 고려하세요.

이상의 내용을 반영하여 YAGNI 원칙에 맞게 리팩토링을 완료하고, 코드를 꼭 필요한 만큼만 남기도록 다듬어주세요.
